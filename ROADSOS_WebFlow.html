What is ROADSOS?

ROADSOS is a full-stack emergency response web & mobile application that connects road accident victims, general users, and emergency responders in real time. It features a user-facing emergency portal, an accident simulation game for demo/testing (which auto-triggers a real SOS alert on crash), and role-based dashboards for Doctors, Police, Hospital staff, and Super Admins.

Token Management

🔑 Access Token — Stored as roadsos_access_token in localStorage. Sent in every API request Authorization header.

🔄 Auto-Refresh — If access token expires, app silently calls POST /auth/refresh in the background and updates stored token.

🚪 Logout — Calls POST /auth/logout, clears localStorage, resets auth state, user returned to home.

Protected Route Guard

Every protected page runs isLoading + user check via the useAuth() hook on mount.

If user is not authenticated after loading, they are auto-redirected to /login.

Role-based components are conditionally rendered — a Police user sees only the Police dashboard even if they navigate directly to /dashboard.

SOS Alert System Flow
How emergencies are triggered, transmitted, and managed by admins
05 / 07
Method A — Manual SOS (Services Page)
1

User Opens /services

Logged-in user sees a red "Trigger SOS" button at the top of the emergency services page.

2

Location Requested

Browser asks for GPS permission. Geolocation API fetches exact latitude & longitude in real time.

3

Modal + Description

A modal appears. User describes the emergency. Address is auto-reverse geocoded via Nominatim API.

4

POST /api/sos

Alert payload sent with: userId, latitude, longitude, address, description, timestamp.

5

Live on Admin Dashboards

All admin dashboards poll every 2 seconds. New SOS appears instantly with map pin + PENDING badge.

Method B — Auto SOS (Game Crash)
1

User Clicks "Demo"

Screen locks to landscape orientation. 3D car simulator loads at /game/index.html.

2

Car Crashes in Simulator

Game engine detects collision. GameOver.jsx fires immediately and redirects to /dashboard?accident=true&speed=X&overtakes=Y.

3

10-Second Countdown

Dashboard shows full-screen red warning + countdown. User can cancel within 10 seconds to abort the SOS.

4

Auto POST /api/sos

If timer hits 0, SOS auto-fires with game data: SIMULATED_CRASH description, crash speed, overtake count, GPS coords.

5

Broadcast to All Admins

Doctor, Police, and Super Admin dashboards all receive the live alert. Screen auto-unlocks to portrait mode.

Demo Game — Crash Simulation Details
Entry: "Demo" button on Home page (logged-in users only)
Orientation: Phone/browser auto-locks to landscape via Capacitor
Game Engine: 3D WebGL car simulator — day/night/evening cycle, traffic
Controls: Arrow keys / on-screen buttons for steering & gas
Crash Detected: Instant redirect to /dashboard?accident=true
Countdown: 10-second full-screen flashing red warning in dashboard
Auto SOS: If not cancelled → auto POST with speed, overtakes, GPS
Admin Alert: Doctor + Police + Super Admin dashboards receive live alert
